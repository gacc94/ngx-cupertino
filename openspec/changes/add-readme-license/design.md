## Decisions

### 1. README Structure

The README follows a proven structure used by major Angular libraries (Angular Material, PrimeNG, Spartan UI):

1. **Header**: Project name + one-liner description
2. **Badges**: Visual status indicators (Angular, TS, Nx, Bun, Vitest, Biome, MIT)
3. **Features**: Key differentiators (Apple Design System, 37 components, Signal API, host-based styling)
4. **Installation**: `bun add @ngx-cupertino/ui` command
5. **Quick Start**: Minimal working example with `provideCupertino()` + `<button cup-button>`
6. **Components**: Full table with 37 components grouped by phase, each showing selector
7. **Architecture**: Package dependency flow diagram (ASCII art or mermaid-compatible)
8. **Conventions**: Styling pattern, commit format, naming conventions
9. **Development**: Common commands for local development (`bun nx serve playground`, `bun nx test ui`, etc.)

### 5. Package Architecture Diagram

Plain ASCII diagram showing dependency flow:

```
@ngx-cupertino/tokens  ←─── @ngx-cupertino/core  ←─── @ngx-cupertino/ui
                                                      ─── @ngx-cupertino/icons
```

- `tokens`: CSS custom properties (no dependencies)
- `core`: Providers, services, directives (depends on `tokens`)
- `icons`: Lucide wrapper (depends on `core`)
- `ui`: All 37 components (depends on `tokens`, `core`, `icons`)

### 6. Quick Start Code

Minimal working example:

```typescript
// app.config.ts
import { provideCupertino } from '@ngx-cupertino/core';

export const appConfig: ApplicationConfig = {
    providers: [provideCupertino()],
};
```

```scss
// styles.scss
@use '@ngx-cupertino/tokens';
```

```html
<!-- component template -->
<button cup-button variant="liquid-glass">
    <cup-icon name="star" cup-prefix />
    Add to Favorites

```

## Risks / Trade-offs

| Risk | Mitigation |
|---|---|
| README component table becomes outdated as components are added | Mark pending components clearly with ⬜. Update table after each phase is complete |
| Badge version numbers become stale | Use major.minor only for framework versions (Angular 21, TS 5.9) to minimize update frequency |
| Consumer copies code that doesn't work yet (components not implemented) | Quick Start uses only `provideCupertino()` and `cup-icon` — choose components that exist or mark examples as upcoming |
| MIT LICENSE year needs annual update | Use `startYear-present` format (e.g., `2026-present`) to auto-extend |
